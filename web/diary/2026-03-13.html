🔧 主要工作内容

1. GitHub Pages 部署问题

早上开始准备将个人主页部署到 GitHub Pages，但遇到了很多问题：

  • index.html 文件推送后显示为空白
  • GitHub 仓库中的文件与本地不一致
  • 多次推送都无法正确更新
  • 网络连接问题导致推送失败

尝试了各种方法：强制推送、重新创建文件、检查 Git 状态，但问题依然存在。这个过程非常煎熬，一度想要放弃。

2. 语言切换功能修复

发现语言切换功能存在问题：

  • 点击语言按钮后页面内容没有正确切换
  • active 状态没有正确更新
  • localStorage 保存的语言偏好没有生效

修复方案：

  • 为每个语言按钮添加 data-lang 属性
  • 使用属性选择器准确识别活动按钮
  • 优化 switchLang 函数逻辑
  • 添加页面加载时恢复语言偏好的功能

3. 移动端适配优化

针对手机端进行了全面的响应式优化：

  • Hero 区域：头像缩小（140px → 100px）、标题字体调整（48px → 32px）
  • 语言切换器：自动换行、按钮缩小、支持触摸操作
  • 开发日记卡片：统计数据从 3 列改为 1 列、按钮全宽显示
  • 项目卡片：优化内边距、字体大小调整
  • 时间线：调整左侧线条位置、圆点缩小
  • 触摸优化：最小点击区域 44px、禁用 hover 效果、添加点击反馈

💡 遇到的问题与解决

问题 1：GitHub 推送失败

现象：git push 命令执行后显示成功，但 GitHub 上的文件没有更新。

原因：网络连接不稳定，GnuTLS 报错。

解决：多次重试、使用强制推送、等待网络恢复。

问题 2：语言切换不生效

现象：点击语言按钮后，页面内容没有切换。

原因：event.target 可能指向按钮内的子元素，导致无法正确识别。

解决：使用 data-lang 属性，通过属性选择器准确识别按钮。

问题 3：移动端显示错乱

现象：手机上布局混乱、字体过大、按钮难以点击。

原因：缺少响应式设计和移动端优化。

解决：添加多个媒体查询断点、优化字体大小、增加触摸区域。
