## 🚀 部署方法

### 方法 1：使用 GitHub CLI（推荐）

```bash
cd /home/admin/openclaw/workspace/projects/guangchu

# 1. 安装 GitHub CLI（如果未安装）
sudo apt update && sudo apt install gh

# 2. 认证 GitHub
gh auth login

# 3. 推送构建目录到 gh-pages 分支
git subtree push --prefix build origin gh-pages
```

### 方法 2：手动推送

```bash
cd /home/admin/openclaw/workspace/projects/guangchu

# 1. 创建临时分支
git checkout --orphan gh-pages-temp

# 2. 添加构建文件
git add -f build/

# 3. 提交
git commit -m "🦞 Deploy diary pages"

# 4. 分离构建目录到新分支
git subtree split --prefix build -b gh-pages

# 5. 推送
git push origin gh-pages --force

# 6. 清理
git checkout main
git branch -D gh-pages-temp
git branch -D gh-pages
```

### 方法 3：使用 GitHub Actions（自动化）

创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Build
        run: |
          chmod +x build-diary-pages.sh
          ./build-diary-pages.sh
      
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./build
```

## ⚠️ 注意事项

### 1. GitHub Pages 设置

在 GitHub 仓库设置中：
1. 进入 **Settings** → **Pages**
2. Source 选择 **gh-pages** 分支
3. Folder 选择 **/ (root)**
4. 点击 **Save**

### 2. 等待生效

部署后需要等待 **1-2 分钟** GitHub Pages 才会更新。

### 3. 清除缓存

如果仍然 404，尝试：
- 强制刷新：Ctrl+Shift+R（Windows）或 Cmd+Shift+R（Mac）
- 清除浏览器缓存
- 使用无痕模式访问

## 🆘 故障排查

### Q1: 推送失败（认证错误）

**解决：**
```bash
# 使用 token 认证
gh auth login
# 或者手动设置 token
export GITHUB_TOKEN=your_token_here
```

### Q2: 404 错误持续存在

**检查：**
1. 确认 gh-pages 分支存在：`git branch -r`
2. 确认 GitHub Pages 设置正确
3. 等待 5 分钟让 GitHub 处理
4. 检查构建文件是否完整

### Q3: 页面显示但样式错误

**解决：**
```bash
# 检查文件路径
ls -la build/

# 确保 CSS/JS 文件路径正确
# 使用相对路径而非绝对路径
```
