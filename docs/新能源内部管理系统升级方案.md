## 🎯 NocoBase 核心借鉴点

### 1. 数据模型驱动（Data Model-Driven）

**核心理念：** UI 与数据结构完全解耦

```
传统方式：
表单/表格 → 绑定数据 → 固定展示

NocoBase 方式：
数据模型 → 多个块（Blocks）→ 多种展示（表格/表单/图表/日历）
```

**光储龙虾应用：**
- 定义核心数据模型：新闻、公司、政策、项目、投资数据
- 同一数据源支持多种视图：列表、看板、日历、地图、图表
- 新增视图无需修改数据结构

### 2. AI 员工（AI Employees）

**核心理念：** AI 不是聊天机器人，而是嵌入业务系统的智能同事

**光储龙虾 AI 员工矩阵：**

| AI 角色 | 职责 | 嵌入位置 |
|--------|------|----------|
| 📰 **行业分析师** | 自动分析新闻，提取关键信息 | 新闻列表页 |
| 📊 **数据科学家** | 分析投资数据，生成评估报告 | 投资地图页 |
| 📝 **内容整理员** | 整理内容，自动打标签分类 | 内容管理页 |
| 🔍 **市场研究员** | 监控竞品动态，生成情报简报 | 市场分析页 |
| 💬 **多语言翻译官** | 自动翻译内容到 5 种语言 | 全局 |
| 📧 **邮件秘书** | 总结邮件，协助回复 | 邮件管理页 |
| 📈 **报告生成器** | 自动生成日报/周报/月报 | 报告中心 |

### 3. 插件化架构（Plugin-Based）

**核心理念：** 一切皆插件，类似 WordPress

```
光储龙虾插件结构：

核心层（Core）
├── 数据模型管理
├── 用户权限系统
├── API 网关
└── 插件管理器

业务插件（Plugins）
├── 新闻抓取插件
├── 投资评估插件
├── 日记管理插件
├── 报告生成插件
├── 邮件管理插件
└── 数据可视化插件

AI 插件（AI Plugins）
├── AI 员工管理
├── 智能分析插件
├── 自动翻译插件
└── 内容生成插件
```

### 4. 块（Blocks）+ 动作（Actions）系统

**核心理念：** 页面由块组成，块通过动作交互

```
页面（Page）
├── 块 1：新闻列表（表格视图）
│   ├── 动作：查看详情
│   ├── 动作：编辑
│   ├── 动作：AI 分析
│   └── 动作：导出
├── 块 2：新闻详情（表单视图）
│   ├── 动作：保存
│   ├── 动作：发布
│   └── 动作：关联项目
├── 块 3：相关新闻（列表视图）
└── 块 4：数据图表（可视化视图）
```

### 5. 事件流（Event Flow）

**核心理念：** 可视化编排业务逻辑

```
事件流示例：新闻抓取后自动处理

触发器：新闻抓取完成
  ↓
条件：新闻重要性 > 阈值
  ↓
动作 1：AI 员工分析内容
  ↓
动作 2：自动打标签
  ↓
动作 3：发送通知给相关人员
  ↓
动作 4：更新投资地图数据
  ↓
动作 5：生成简报
```

### 6. RUNJS（自定义代码扩展）

**核心理念：** 在页面中直接写代码扩展功能

```javascript
// 示例：新闻列表页自定义字段
// 在页面设计器中直接编写

return {
  title: news.title,
  sentiment: analyzeSentiment(news.content),
  relatedCompanies: extractCompanies(news.content),
  investmentScore: calculateScore(news, marketData)
};
```

                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      AI 服务层（AI Services）                  │
├─────────────────────────────────────────────────────────────┤

│  ─────────────────────────────────────────────────────────  │
│           LLM 接口（Qwen/Claude/GPT）+ RAG 引擎              │

                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      业务层（Business Logic）                 │
├─────────────────────────────────────────────────────────────┤
│  新闻管理  │  投资评估  │  项目管理  │  报告生成  │  工作流   │
│  ─────────────────────────────────────────────────────────  │
│                    插件系统（Plugin System）                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      数据层（Data Layer）                     │
├─────────────────────────────────────────────────────────────┤

### 核心数据模型

```yaml
# 新闻（News）
- id: UUID
- title: String
- content: Text
- source: String (信息源)
- publishDate: DateTime
- language: String (zh/en/ja/fr/es)
- tags: Array (AI 自动生成)
- sentiment: Float (情感分析)
- companies: Array (关联公司)
- countries: Array (关联国家)
- topics: Array (主题分类)
- importance: Integer (重要性评分)
- aiSummary: Text (AI 摘要)
- createdAt: DateTime
- updatedAt: DateTime

# 公司（Company）
- id: UUID
- name: String
- country: String
- industry: String (光伏/储能/电力)
- marketCap: Float
- stockCode: String
- description: Text
- news: Relation (关联新闻)
- projects: Relation (关联项目)

# 项目（Project）
- id: UUID
- name: String
- type: String (电站/制造/研发)
- country: String
- province: String
- capacity: Float (MW)
- investment: Float (USD)
- status: String (规划/建设/运营)
- companies: Relation (关联公司)
- timeline: Array (时间节点)

# 投资评估（InvestmentEvaluation）
- id: UUID
- country: String
- province: String
- score: Float (综合评分)
- factors: JSON (评估因子)
- risks: Array (风险列表)
- opportunities: Array (机会列表)
- lastUpdated: DateTime

# AI 员工（AIEmployee）
- id: UUID
- name: String
- role: String (角色)
- skills: Array (技能列表)
- tools: Array (工具调用权限)
- knowledgeBase: Array (知识库)
- conversationHistory: Relation (对话历史)
- performance: JSON (表现统计)
```

## 🛠️ 技术实现路线

### 方案 A：基于 NocoBase 二次开发（推荐）

**优势：**
- ✅ 直接使用成熟的无代码/低代码平台
- ✅ 内置 AI 员工系统
- ✅ 插件化架构现成
- ✅ 多应用/多空间支持
- ✅ 节省 70% 开发时间

**实施步骤：**

1. **部署 NocoBase**
   ```bash
   # Docker 部署
   docker run -d --name nocobase \
     -p 13000:80 \
     -v nocobase-storage:/app/nocobase/storage \
     nocobase/nocobase:latest
   ```

2. **定制数据模型**
   - 在 NocoBase 中定义新闻、公司、项目等模型
   - 配置字段类型和关系

3. **开发定制插件**
   - 新闻抓取插件（Python → Node.js 适配）
   - 投资评估插件
   - AI 员工技能插件

4. **配置 AI 员工**
   - 定义各角色 AI 员工
   - 配置工具和知识库
   - 设置触发条件

5. **UI 定制**
   - 使用页面设计器定制界面
   - 添加 RUNJS 自定义逻辑
   - 配置 Event Flow

**成本估算：**
- 时间：2-3 个月
- 人力：1-2 名开发者
- 费用：NocoBase 商业插件（一次性付费）

### 方案 B：参考 NocoBase 自研

**优势：**
- ✅ 完全自主可控
- ✅ 深度定制新能源场景
- ✅ 无外部依赖

**挑战：**
- ❌ 开发周期长（6-12 个月）
- ❌ 需要完整团队（前端/后端/AI/测试）
- ❌ 维护成本高

**技术栈建议：**

```
前端：
- React + TypeScript
- FlowEngine 自研（参考 NocoBase）
- ECharts / Ant Design Charts

后端：
- Node.js + Koa / NestJS
- PostgreSQL + SQLite FTS5
- Redis（缓存）
- Bull（任务队列）

AI：
- LLM API（Qwen/Claude/GPT）
- LangChain / LlamaIndex
- RAG 引擎
- Vector DB（Pinecone/Milvus）

部署：
- Docker + Kubernetes
- CI/CD（GitHub Actions）
- 监控（Prometheus + Grafana）
```

**成本估算：**
- 时间：6-12 个月
- 人力：3-5 名开发者
- 费用：服务器 + API 调用

## 🎯 推荐方案

### 首选：方案 A（基于 NocoBase 二次开发）

**理由：**
1. **时间最短** - 2-3 个月上线 MVP
2. **成本最低** - 首年仅需~$2000
3. **风险最小** - 成熟平台，经过验证
4. **AI 原生** - 内置 AI 员工系统，无需从零开发
5. **可扩展** - 插件化架构，后续可逐步替换自研模块

**实施建议：**
1. 立即部署 NocoBase 测试环境
2. 导入现有数据模型（新闻/公司/项目）
3. 迁移现有 Python 脚本为插件
4. 配置 AI 员工（先上线 2-3 个核心角色）
5. 小范围试点，收集反馈
6. 迭代优化，逐步扩展

### 备选：方案 C（混合方案）

**适用场景：**
- NocoBase 某些功能无法满足需求
- 需要深度定制 AI 能力
- 有特定性能要求
